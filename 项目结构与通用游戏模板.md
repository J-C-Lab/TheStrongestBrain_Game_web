# 🧩 React 益智游戏项目全局上下文 (AI Sync Template)

## 1. 核心技术栈 (Core Stack)
- **框架**: React 18 (Vite 构建)
- **样式**: Tailwind CSS (Apple 设计语言)
- **路由**: `react-router-dom` (URL 参数驱动: `/game?id=xxx`)
- **交互**: 具备“卡包抽卡”悬浮动画，图标从容器背后向上滑出

---

## 2. 游戏通用开发规范 (Global Game Standards)

### 🕒 核心状态管理 (Core States)
- **游戏状态 (`gameState`)**: `playing` | `won` | `lost` | `surrendered`
- **计时系统**: `timer`: 记录 `seconds`，由 `useEffect` 在挂载时开启，计入`timeSpent`
- **游戏积分 (`score`)**: `score`: 根据 `difficulty` 倍率或 `timer` 计算实时积分。
- **难度逻辑**: 
   **难易程度difficulty**
  - **简单**: 系数 1.0 (5分/组)
  - **中等**: 系数 2.0 (10分/组)
  - **困难**: 系数 4.0 (20分/组)
  或**总游戏时长timer**
  - **以下10分钟**: 20分
  - **10-20分钟**: 10分
  - **20分钟以上**: 5分

### 🛠️ 必备通用 UI 组件 (Common UI Components)
- **🚩 顶部信息栏**:
  - `关卡标题`: 显示当前游戏关卡信息
  - `实时计时器`: 显示当前游戏时长
  - `规则按钮`: 点击/悬浮显示当前游戏规则弹窗
  - `投降按钮`: 点击后清除积分并上报失败状态
- **投降逻辑**: 包含 `Surrender` 按钮，触发 `gameState = 'lost'`，点击后清除积分并上报失败状态
 **🎊 结算反馈 (Result Feedback)**:
  - `胜利`: 触发 `canvas-confetti` 彩带效果，显示“挑战成功”与本轮所加分数弹窗，提供“结算并离开”和“下一关”选项。
  - `失败`: 震动/变暗反馈，显示“挑战失败”语录，提供“重新开始”/“返回仓库”选项。
